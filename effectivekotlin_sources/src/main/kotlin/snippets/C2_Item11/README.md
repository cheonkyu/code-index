# Item 11: 가독성을 높이는 디자인

안녕하세요! 코틀린 프로그래밍 전문가이자 친절한 강사입니다. Effective Kotlin 책의 11번째 아이템인 "가독성을 높이는 디자인"에 대해 쉽고 친절하게 설명해 드릴게요. 코드를 작성하는 것은 단순히 컴퓨터에게 명령을 내리는 것이 아니라, 다른 개발자(혹은 미래의 자신!)에게 내용을 전달하는 과정이기도 합니다. 따라서 코드를 얼마나 쉽게 이해할 수 있게 작성하느냐가 중요하며, 이것이 바로 가독성입니다.

## 왜 가독성이 중요할까요?

가독성이 높은 코드는 다음과 같은 장점을 가져다 줍니다.

* **유지보수 용이:** 코드를 이해하기 쉬우므로 버그 수정이나 기능 추가가 훨씬 간편해집니다.
* **협업 효율 증대:** 다른 개발자와 함께 작업할 때, 코드를 빠르게 이해하고 협업할 수 있습니다.
* **오류 감소:** 코드를 명확하게 작성하면 실수를 줄이고 예상치 못한 동작을 방지할 수 있습니다.
* **시간 절약:** 코드를 읽고 이해하는 데 드는 시간을 줄여 전체 개발 속도를 높일 수 있습니다.

## 가독성을 높이기 위한 핵심 가이드라인

Effective Kotlin에서는 가독성을 높이기 위해 다음과 같은 가이드라인을 제시합니다.

1. **의미 있는 이름을 사용하세요.** 변수, 함수, 클래스 등의 이름을 지을 때, 그 역할이나 목적을 명확하게 나타내는 이름을 선택해야 합니다. 추상적이거나 모호한 이름은 피하는 것이 좋습니다.

   ```kotlin
   // 나쁜 예
   val x = 10
   val y = 20

   // 좋은 예
   val numberOfStudents = 10
   val maxCapacity = 20
   ```

   `x` 와 `y` 만으로는 어떤 값을 나타내는지 알 수 없지만, `numberOfStudents` 와 `maxCapacity` 는 그 의미를 명확하게 전달합니다.

2. **간결하게 작성하세요.** 불필요하게 복잡하거나 장황한 코드는 가독성을 떨어뜨립니다. 코드를 최대한 간결하게 유지하고, 중복되는 코드는 제거해야 합니다. 코틀린의 기능들을 적극 활용하여 코드를 간결하게 만들 수 있습니다.

3. **일관성을 유지하세요.** 코드 스타일, 들여쓰기, 주석 등의 규칙을 정하고 일관되게 적용해야 합니다. 일관성이 없는 코드는 읽기가 어렵고 혼란을 야기할 수 있습니다. 코틀린 공식 스타일 가이드를 참고하는 것도 좋은 방법입니다. ([https://kotlinlang.org/docs/coding-style.html](https://kotlinlang.org/docs/coding-style.html))

4. **주석을 적절하게 사용하세요.** 코드가 복잡하거나 이해하기 어려운 경우, 주석을 사용하여 설명을 추가할 수 있습니다. 하지만 주석은 코드를 대체할 수 없으므로, 코드 자체가 명확하도록 작성하는 것이 우선입니다. 주석은 코드의 *왜* 를 설명하는 데 사용하는 것이 좋습니다.

5. **함수와 클래스를 작게 유지하세요.** 함수나 클래스가 너무 크면 이해하기 어렵고 재사용성이 떨어집니다. 하나의 함수나 클래스는 하나의 책임만 갖도록 설계하고, 필요한 경우 더 작은 단위로 분리해야 합니다.

## 코드 예제: 가독성을 향상시킨 리팩토링

다음은 가독성이 낮은 코드를 리팩토링하여 가독성을 높인 예제입니다.

```kotlin
// 가독성이 낮은 코드
fun calculate(a: Int, b: Int, op: String): Int {
    if (op == "+") return a + b
    if (op == "-") return a - b
    if (op == "*") return a * b
    if (op == "/") {
        if (b == 0) throw IllegalArgumentException("0으로 나눌 수 없습니다.")
        return a / b
    }
    throw IllegalArgumentException("지원하지 않는 연산자입니다.")
}

// 가독성이 높은 코드
enum class Operation {
    ADD, SUBTRACT, MULTIPLY, DIVIDE
}

fun calculate(a: Int, b: Int, operation: Operation): Int {
    return when (operation) {
        Operation.ADD -> a + b
        Operation.SUBTRACT -> a - b
        Operation.MULTIPLY -> a * b
        Operation.DIVIDE -> {
            if (b == 0) throw IllegalArgumentException("0으로 나눌 수 없습니다.")
            a / b
        }
    }
}
```

가독성이 낮은 코드는 `if` 문을 사용하여 연산자를 비교하고 있습니다. 반면, 가독성이 높은 코드는 `enum` 클래스를 사용하여 연산자를 정의하고, `when` 문을 사용하여 각 연산에 대한 처리를 수행합니다. 이렇게 하면 코드의 가독성이 훨씬 높아지고, 유지보수도 용이해집니다.

## 핵심 요약

* 가독성이 높은 코드는 유지보수, 협업, 오류 감소, 시간 절약에 도움이 됩니다.
* 의미 있는 이름 사용, 간결한 코드 작성, 일관성 유지, 적절한 주석 사용, 작은 함수/클래스 유지 등을 통해 가독성을 높일 수 있습니다.
* 코드의 가독성을 높이기 위해 코틀린의 기능을 적극적으로 활용하세요.
* 가독성은 단순히 코드를 보기 좋게 만드는 것을 넘어, 코드의 품질과 생산성을 향상시키는 중요한 요소입니다.
