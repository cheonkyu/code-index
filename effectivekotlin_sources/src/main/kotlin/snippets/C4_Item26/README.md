# Item 26: 단일 추상화 수준으로 함수 작성하기

안녕하세요! 오늘은 코틀린 프로그래밍에서 매우 중요한 원칙 중 하나인 **단일 추상화 수준으로 함수 작성하기**에 대해 알아보겠습니다. 이 원칙은 코드의 가독성과 유지보수성을 크게 향상시키는 데 도움이 됩니다.

## 핵심 가이드라인 및 이유

### 가이드라인
- **함수 하나당 하나의 추상화 수준**: 각 함수는 하나의 명확한 추상화 수준에 집중해야 합니다. 즉, 함수의 단일 목적은 하나의 개념이나 문제 해결 방식에만 초점을 맞추는 것이 좋습니다.

### 이유
- **가독성 향상**: 하나의 추상화 수준에 집중하면 함수의 목적이 명확해져 코드를 이해하는 데 도움이 됩니다.
- **유지보수 용이성**: 코드 변경이나 확장이 필요할 때, 특정 추상화 수준에 집중되어 있는 함수는 더 쉽게 수정할 수 있습니다.
- **재사용성 증대**: 단일 목적에 특화된 함수는 특정 상황에서 더 효과적으로 재사용될 수 있습니다.

## 예제 코드

다음은 이 원칙을 적용한 간단한 예제입니다. 여기서는 사용자 정보를 처리하는 두 개의 함수를 비교해 보겠습니다.

### 나쁜 예제 (다중 추상화 수준)
```kotlin
fun processUserData(user: User) {
    val name = user.name ?: ""
    val age = user.age ?: 0
    println("이름: $name, 나이: $age")
}
```
이 함수는 사용자 이름과 나이를 처리하지만, 이름과 나이 모두를 다루는 복잡한 로직이 혼재되어 있어 이해하기 어렵습니다.

### 좋은 예제 (단일 추상화 수준)
```kotlin
// 이름만 다루는 함수
fun printUserName(user: User): String {
    return user.name ?: ""
}

// 나이만 다루는 함수
fun printUserAge(user: User): Int {
    return user.age ?: 0
}

// 결과 출력
fun displayUserInfo(user: User) {
    println("이름: ${printUserName(user)}, 나이: ${printUserAge(user)}")
}
```
이 예제에서는 각각의 함수가 단일 추상화 수준, 즉 이름 처리와 나이 처리에만 집중하고 있습니다. 이를 통해 각 함수의 역할이 명확해지고 코드가 더 직관적으로 이해됩니다.

## 핵심 요약
- 함수는 하나의 명확한 추상화 수준에 집중해야 합니다.
- 이는 코드 가독성과 유지보수성을 크게 향상시킵니다.
- 예제를 통해 단일 목적 함수의 효과를 확인할 수 있습니다.
