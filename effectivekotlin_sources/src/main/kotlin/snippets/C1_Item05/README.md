# Item 5: 인자의 기대값과 상태를 명확하게 지정하세요

안녕하세요! 코틀린 프로그래밍을 공부하시는 여러분을 위해 Effective Kotlin의 Item 5를 쉽고 자세하게 설명해 드리려고 합니다. 이 아이템은 함수를 설계할 때 인자의 기대값과 함수의 상태 변화를 명확하게 지정하는 것이 왜 중요한지, 그리고 어떻게 하면 효과적으로 이를 실천할 수 있는지 다룹니다.

## 왜 중요할까요? (Best Practice)

함수를 사용하는 사람은 함수가 어떤 종류의 인자를 기대하고, 함수를 실행한 후 함수의 상태가 어떻게 변할지 정확히 알고 싶어합니다. 이러한 정보가 명확하지 않으면, 함수를 오용하거나 예상치 못한 동작이 발생할 수 있습니다. 명확한 지정은 다음의 이점을 가져다 줍니다.

* **가독성 향상:** 함수를 읽는 사람이 코드를 더 쉽게 이해하고 유지보수할 수 있습니다.
* **오류 감소:** 잘못된 인자를 전달하거나 함수의 상태 변화를 잘못 이해하여 발생하는 오류를 줄일 수 있습니다.
* **안정적인 코드:** 함수를 더욱 안정적으로 만들고 예상치 못한 동작을 방지할 수 있습니다.

## 어떻게 하면 좋을까요?

이 아이템에서는 다음과 같은 구체적인 방법을 제시합니다.

1. **Nullable 타입 명시:** 인자가 null 값을 가질 수 있는지 명확히 표현해야 합니다.  만약 null 값을 허용하지 않는다면, `NotNull` 어노테이션 (혹은 기본적으로 null이 아닌 타입)을 사용해서 명시적으로 나타내는 것이 좋습니다.
2. **기본값 활용:**  인자에 적절한 기본값을 제공하면, 함수를 호출하는 쪽에서 불필요한 인자를 생략할 수 있습니다.
3. **함수 상태 변화 명시:** 함수가 내부 상태를 변경하는 경우, 이를 문서화하거나 함수명 자체에 표현하여 사용자가 알 수 있도록 해야 합니다. 예를 들어, 상태를 변경하는 함수는 `update`, `set`, `modify` 와 같은 이름을 사용하는 것이 좋습니다.
4. **불변 객체 활용:**  가능하다면, 함수 내부에서 객체의 상태를 변경하는 대신 새로운 객체를 반환하는 불변 객체를 사용하는 것을 고려해 보세요. 이는 예상치 못한 부작용을 줄이고 코드를 더욱 예측 가능하게 만들어줍니다.

## 예제 코드

```kotlin
// 인자가 null을 허용하지 않는 예시
fun greet(name: String) {
    println("Hello, $name!")
}

// 인자가 null을 허용하고 기본값을 제공하는 예시
fun greetOrGuest(name: String? = "Guest") {
    println("Hello, ${name ?: "Guest"}!")
}

// 상태를 변경하는 함수 예시 (가능하면 피하는 것이 좋습니다)
class Counter {
    var count = 0
    fun increment() {
        count++ // 상태 변경
    }
}

// 불변 객체를 활용한 함수 예시 (권장)
data class CounterState(val count: Int)
fun incrementCounter(state: CounterState): CounterState {
    return CounterState(state.count + 1) // 새로운 상태 객체 반환
}
```

위의 예제 코드를 보면, `greet` 함수는 `name` 인자가 null이 될 수 없도록 설계되었습니다. `greetOrGuest` 함수는 `name` 인자가 null이 될 수 있으며, 기본값으로 "Guest"를 사용합니다. `Counter` 클래스의 `increment` 함수는 내부 상태를 변경하지만, `incrementCounter` 함수는 불변 객체를 활용하여 새로운 상태를 반환합니다.

## 핵심 요약

* 함수 인자의 null 허용 여부를 명확하게 지정하고, 기본값을 활용하여 사용 편의성을 높이세요.
* 함수가 상태를 변경하는 경우, 이를 명확히 문서화하거나 함수명에 반영하세요.
* 가능한 한 불변 객체를 사용하여 상태 변화를 최소화하고, 예측 가능한 코드를 작성하세요.
* 명확한 기대값과 상태 지정은 가독성, 안정성, 유지보수성을 향상시키는 중요한 요소입니다.
