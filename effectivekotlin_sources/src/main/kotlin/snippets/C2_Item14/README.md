# Item 14: 변수 타입이 불분명할 때는 타입을 명시하세요

코틀린은 타입 추론이라는 강력한 기능을 제공하기 때문에, 대부분의 경우 변수 선언 시 타입을 명시하지 않아도 코틀린 컴파일러가 알아서 타입을 추론해줍니다. 하지만, 변수 타입이 명확하지 않거나, 코드의 가독성을 높이기 위해서는 변수 타입을 명시하는 것이 좋습니다. 이 아이템에서는 언제 변수 타입을 명시해야 하는지, 그리고 그 이유는 무엇인지 알아보겠습니다.

## 왜 타입을 명시해야 할까요?

타입 추론은 편리하지만, 복잡한 표현식이나 상황에서는 컴파일러가 어떤 타입을 추론해야 할지 헷갈릴 수 있습니다. 이런 경우, 명시적인 타입 선언은 다음과 같은 장점을 제공합니다.

*   **가독성 향상:** 변수 타입을 명시하면 코드를 읽는 사람이 변수가 어떤 값을 저장할 것인지 더 쉽게 이해할 수 있습니다.
*   **오류 방지:** 타입 추론이 예상과 다른 타입으로 이루어지는 경우, 런타임 오류가 발생할 수 있습니다. 타입을 명시하면 이러한 오류를 컴파일 시점에 발견할 수 있습니다.
*   **예상치 못한 타입 변경 방지:** 코드를 수정하면서 타입 추론이 변경되어 의도하지 않은 동작이 발생할 수 있습니다. 명시적인 타입 선언은 이러한 상황을 방지합니다.

## 언제 타입을 명시해야 할까요?

다음과 같은 경우에 변수 타입을 명시하는 것이 좋습니다.

1.  **변수 초기화 값이 없는 경우:** 변수를 선언만 하고 나중에 초기화하는 경우, 코틀린은 변수의 타입을 추론할 수 없습니다. 이럴 때는 반드시 타입을 명시해야 합니다.

    ```kotlin
    val name: String // 초기화 값이 없으므로 타입 명시
    name = "Kotlin"
    ```

2.  **복잡한 표현식에서 추론된 타입이 불명확한 경우:** 복잡한 계산이나 함수 호출 결과가 변수에 할당될 때, 컴파일러가 어떤 타입을 추론해야 할지 명확하지 않을 수 있습니다.

    ```kotlin
    val result = someFunction() // someFunction()의 반환 타입이 불명확할 경우
    val result: Int = someFunction() // 명시적인 타입 선언
    ```

3.  **가독성을 위해:** 변수 타입이 명확하더라도, 코드를 읽는 사람의 이해를 돕기 위해 타입을 명시하는 것이 좋습니다. 특히, 도메인 지식과 관련된 변수의 경우, 타입을 명시하면 코드의 의미를 더 쉽게 파악할 수 있습니다.

    ```kotlin
    val userAge: Int = 30 // 나이는 정수라는 정보를 명확히
    val userName: String = "Alice" // 이름은 문자열이라는 정보를 명확히
    ```

4. **null 허용 여부를 명확하게 하고 싶을 때:**  nullable 타입인지 non-nullable 타입인지 명확하게 하고 싶을 때는 타입을 명시적으로 지정하는 것이 좋습니다.

   ```kotlin
   val address: String? = null // nullable String 타입 명시
   val city: String = "Seoul"   // non-nullable String 타입 명시
   ```

## 타입 추론을 활용하는 경우

타입 추론은 코드를 간결하게 만들어주는 강력한 기능입니다. 타입이 명확한 경우에는 타입 추론을 적극적으로 활용하여 코드의 중복을 줄이고 가독성을 높일 수 있습니다.

```kotlin
val age = 30 // 타입 추론: Int
val message = "Hello, Kotlin!" // 타입 추론: String
```

## 핵심 요약

*   변수 타입이 불분명하거나 가독성을 높이기 위해서는 변수 타입을 명시하세요.
*   변수 초기화 값이 없는 경우, 복잡한 표현식에서 추론된 타입이 불명확한 경우, 그리고 가독성을 위해 타입 명시가 필요합니다.
*   타입 추론은 코드를 간결하게 만들어주지만, 때로는 명시적인 타입 선언이 더 안전하고 이해하기 쉬운 코드를 만들어줍니다.
*   Nullable/Non-nullable 타입을 명확하게 구분하여 오류를 방지하세요.
