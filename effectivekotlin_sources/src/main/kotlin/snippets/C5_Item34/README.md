# Item 34: 명명된 선택적 매개변수를 가진 주 생성자 고려하기

## 핵심 가이드라인 및 이유 (Best Practice)

클래스의 주 생성자를 사용할 때, 모든 매개변수를 필수적으로 만들 필요는 없습니다.  특히 클래스가 여러 개의 설정 옵션을 가지고 있다면, 모든 옵션을 한 번에 제공해야 하는 부담을 줄이기 위해 **명명된 선택적 매개변수**를 사용하는 것이 좋습니다.

명명된 선택적 매개변수는 다음과 같은 장점을 제공합니다.

*   **가독성 향상:** 매개변수 이름을 명시적으로 사용하므로, 어떤 값이 어떤 매개변수에 해당하는지 쉽게 알 수 있습니다.
*   **유연성 증가:** 필요한 매개변수만 전달하여 객체를 생성할 수 있습니다.  기본값을 설정해두면 생략해도 문제가 없습니다.
*   **API의 일관성 유지:** 클래스의 생성 규칙을 명확하게 정의하여 API 사용에 혼란을 줄입니다.
*   **코드 유지보수 용이성:**  매개변수 추가 시 기존 코드에 큰 영향을 주지 않습니다.

## 예제 코드

다음은 명명된 선택적 매개변수를 사용하여 `Person` 클래스를 정의하는 예제입니다.

```kotlin
data class Person(
    val name: String,
    val age: Int = 0, // 기본값 0
    val city: String? = null // 기본값 null
)

fun main() {
    val person1 = Person("Alice") // 이름만 전달
    println(person1) // Person(name=Alice, age=0, city=null)

    val person2 = Person("Bob", age = 30) // 이름과 나이 전달
    println(person2) // Person(name=Bob, age=30, city=null)

    val person3 = Person("Charlie", city = "Seoul") // 이름과 도시 전달
    println(person3) // Person(name=Charlie, age=0, city=Seoul)

    val person4 = Person("David", age = 25, city = "Busan") // 모든 값 전달
    println(person4) // Person(name=David, age=25, city=Busan)
}
```

위 예제에서 `age`와 `city`는 선택적 매개변수이며, 기본값이 설정되어 있습니다. 따라서 객체를 생성할 때 해당 매개변수를 생략할 수 있습니다.  매개변수를 지정할 때는 `매개변수 이름 = 값` 형태로 명시적으로 지정해야 합니다.

## 추가 설명

*   선택적 매개변수에는 기본값을 지정해야 합니다. 기본값은 해당 매개변수가 생략되었을 때 사용될 값입니다.
*   `null`을 허용하는 타입의 매개변수는 기본값을 `null`로 설정할 수 있습니다.
*   주 생성자 외에 부 생성자를 사용하는 경우에도 명명된 선택적 매개변수를 활용할 수 있습니다.
*   데이터 클래스의 경우, `data` 키워드를 사용하면 `equals()`, `hashCode()`, `toString()` 등의 메서드가 자동으로 생성됩니다.

## 핵심 요약

명명된 선택적 매개변수는 클래스 생성 시 유연성과 가독성을 높이는 효과적인 방법입니다. 필요한 매개변수만 전달하여 객체를 생성하고, 기본값을 통해 생략을 허용할 수 있습니다. 이를 통해 API를 더 쉽게 사용할 수 있도록 만들고, 코드 유지보수성을 향상시킬 수 있습니다. 꼭 활용해서 깔끔하고 유연한 코드를 작성해 보세요!
