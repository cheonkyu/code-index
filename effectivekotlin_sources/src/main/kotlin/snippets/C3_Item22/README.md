# Item 22: 일반 알고리즘 구현 시 제네릭 사용하기

제네릭 프로그래밍은 코드의 유연성과 재사용성을 극대화하는 중요한 기법입니다. 특히 공통 알고리즘을 구현할 때 제네릭 타입을 사용하면 다양한 데이터 타입에 대해 동일한 로직을 적용할 수 있습니다. 이는 코드의 유지보수성을 향상시키고 오류 발생 가능성을 줄이는 데 큰 도움이 됩니다.

## 핵심 가이드라인 (Best Practices)

1. **타입 안전성 확보**: 제네릭을 사용하면 컴파일 시점에 타입 오류를 잡아낼 수 있습니다. 이로 인해 런타임 오류를 미리 방지할 수 있습니다.
   
2. **코드 재사용성 향상**: 동일한 알고리즘을 다양한 데이터 타입에 적용할 수 있으므로, 코드를 반복적으로 작성할 필요가 없어집니다. 이는 개발 시간을 절약하고 코드의 일관성을 유지하는 데 도움이 됩니다.

3. **유지보수성 향상**: 변경 사항이 필요할 때 특정 타입에만 영향을 미치므로 전체 코드베이스에 대한 수정 범위를 최소화할 수 있습니다.

## 예제 코드

다음은 `List`에 있는 요소들을 모두 출력하는 간단한 알고리즘을 제네릭으로 구현한 예제입니다. 이 예제에서는 `Item22GenericFunction`이라는 함수를 정의하여 다양한 타입의 리스트를 처리할 수 있도록 합니다.

```kotlin
// 제네릭 함수 정의
fun <T> printListItems(list: List<T>) {
    for (item in list) {
        println(item)  // 여기서 T는 실제 타입에 따라 적절한 출력 형식으로 확장 가능
    }
}

// 다양한 타입의 리스트 사용 예제
fun main() {
    val intList = listOf(1, 2, 3, 4, 5)
    val stringList = listOf("사과", "바나나", "체리")
    
    println("정수 리스트 출력:")
    printListItems(intList)
    
    println("\n문자열 리스트 출력:")
    printListItems(stringList)
}
```

### 설명
- `<T>`는 타입 매개변수로, 이 경우 `T`는 함수가 처리할 수 있는 어떤 타입이든 될 수 있습니다.
- `printListItems` 함수는 `List<T>`를 입력으로 받아 각 요소를 출력합니다.
- `main` 함수에서는 정수 리스트와 문자열 리스트를 각각 함수에 전달하여 다양한 타입에 대해 동일한 알고리즘을 적용할 수 있음을 보여줍니다.

## 핵심 요약
- 제네릭을 사용하면 타입 안전성과 코드 재사용성이 향상됩니다.
- 다양한 데이터 타입에 대해 동일한 알고리즘을 적용할 수 있어 유지보수성이 높아집니다.
- 이를 통해 효율적이고 유연한 코드를 작성할 수 있습니다.
