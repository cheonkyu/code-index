# Item 37: 데이터 묶음을 표현할 때는 `data` modifier를 사용하세요

안녕하세요! 코틀린 프로그래밍을 배우는 여러분을 위해 Effective Kotlin 책의 Item 37에 대한 내용을 쉽고 자세하게 설명해 드릴게요. 이 아이템은 데이터를 저장하고 관리하는 데 매우 유용한 `data` modifier에 대한 내용입니다.

## 왜 `data` modifier를 사용해야 할까요?

코틀린에서 데이터 클래스는 특정 속성들을 가진 객체를 나타낼 때 사용됩니다. 예를 들어, 사람의 이름, 나이, 주소 등을 하나의 객체로 묶을 수 있겠죠.  이러한 데이터 클래스는 일반적으로 다음과 같은 기능들이 필요합니다.

*   **데이터 저장**: 당연히 데이터를 저장해야겠죠?
*   **데이터 비교**: 두 객체가 같은 데이터를 가지고 있는지 확인해야 합니다.
*   **객체 복사**: 객체의 내용을 복사하여 새로운 객체를 만들어야 할 때가 있습니다.
*   **문자열 표현**: 객체의 내용을 사람이 읽기 쉬운 형태로 출력해야 합니다.

이러한 기능들을 직접 구현하려면 많은 코드를 작성해야 합니다. 하지만 코틀린의 `data` modifier를 사용하면 이러한 기능들을 **자동으로** 제공받을 수 있습니다!  따라서 코드를 더 간결하고 읽기 쉽게 만들 수 있으며, 실수 가능성도 줄어듭니다.

## `data` modifier 사용법

`data` 키워드는 클래스 선언 앞에 붙여서 사용합니다.  아래 예시를 통해 `data` 클래스를 만들어 보겠습니다.

```kotlin
data class User(val name: String, val age: Int)
```

이렇게 간단하게 `data` 클래스를 정의하면, 코틀린 컴파일러가 다음 기능들을 자동으로 생성해 줍니다.

*   **`equals()`**: 두 `User` 객체가 `name`과 `age`가 동일한지 비교합니다.
*   **`hashCode()`**: 객체의 해시 코드를 생성합니다. (`equals()`와 함께 사용되어 HashMap 등에서 객체를 식별하는 데 사용됩니다.)
*   **`toString()`**: 객체의 내용을 사람이 읽기 쉬운 문자열 형태로 출력합니다. (예: `User(name=John, age=30)`)
*   **`copy()`**: 객체의 내용을 복사하여 새로운 `User` 객체를 생성합니다.  필요에 따라 일부 속성만 변경하여 복사할 수도 있습니다.
*   **`componentN()` 함수**:  각 속성에 접근하기 위한 함수들을 생성합니다. (구조적 분해를 지원합니다.)

## 예시 코드

```kotlin
fun main() {
    val user1 = User("Alice", 25)
    val user2 = User("Alice", 25)
    val user3 = User("Bob", 30)

    println(user1 == user2) // true (equals() 사용)
    println(user1 == user3) // false

    println(user1) // User(name=Alice, age=25) (toString() 사용)

    val user4 = user1.copy(age = 26) // age만 변경하여 복사
    println(user4) // User(name=Alice, age=26)

    val (name, age) = user1 // 구조적 분해
    println("Name: $name, Age: $age") // Name: Alice, Age: 25
}
```

위 예시에서 볼 수 있듯이, `data` 클래스를 사용하면 객체 비교, 문자열 출력, 객체 복사 등을 매우 쉽게 할 수 있습니다.

## 주의사항

`data` 클래스는 **불변성**을 지향하는 것이 좋습니다.  `val`을 사용하여 속성을 선언하면 속성 값을 변경할 수 없으므로 데이터의 일관성을 유지하는 데 도움이 됩니다.  하지만 `var`를 사용할 수도 있습니다. 상황에 맞게 선택하되, 데이터의 변경 가능성이 높다면 `data` 클래스보다는 일반 클래스를 사용하는 것을 고려해 보세요.

## 핵심 요약

*   `data` modifier는 데이터 묶음을 표현하는 클래스를 간결하게 정의할 수 있도록 도와줍니다.
*   `equals()`, `hashCode()`, `toString()`, `copy()`, `componentN()` 함수들을 자동으로 생성해 줍니다.
*   데이터 클래스는 불변성을 지향하는 것이 좋으며, `val`을 사용하여 속성을 선언하는 것이 권장됩니다.
*   데이터 클래스를 사용하면 코드를 더 간결하고 유지보수하기 쉽게 만들 수 있습니다.
