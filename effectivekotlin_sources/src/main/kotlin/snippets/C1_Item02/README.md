# Item 2: 변수의 범위 최소화하기

안녕하세요! 오늘은 Kotlin 프로그래밍에서 매우 중요한 주제인 "변수의 범위 최소화하기"에 대해 자세히 알아보겠습니다. 이 원칙은 코드의 가독성과 유지보수성을 크게 향상시키는 데 도움이 됩니다. 초보자분들도 쉽게 이해할 수 있도록 핵심 가이드라인과 실제 예제를 통해 설명해 드리겠습니다.

## 핵심 가이드라인 및 이유 (Best Practice)

### 1. **필요한 곳에서만 변수 사용하기**
   - **목표**: 변수를 필요한 최소한의 범위 내에서만 사용하여 코드의 명확성을 높입니다.
   - **이유**: 변수의 범위를 좁히면 해당 변수가 변경되는 범위가 줄어들어 코드 오류 가능성이 감소하고, 다른 부분에서의 불필요한 의존성이 줄어들어 코드 유지보수가 용이해집니다.

### 2. **지역 변수 우선 사용**
   - **목표**: 함수 내에서만 사용되는 변수는 지역 변수로 선언합니다.
   - **이유**: 지역 변수는 함수가 종료되면 자동으로 소멸되므로 메모리 관리가 효율적입니다. 또한, 함수 내에서만 필요한 정보를 제한적으로 다루어 코드의 복잡성을 줄입니다.

### 3. **클래스 멤버 변수 최소화**
   - **목표**: 클래스 내에서도 멤버 변수 사용을 최소화하고, 필요한 경우에만 사용합니다.
   - **이유**: 멤버 변수는 클래스 전체에 걸쳐 영향을 미치므로, 과도한 사용은 클래스의 책임 분리를 어렵게 만들고 테스트와 유지보수를 복잡하게 합니다.

## 예제 코드

### 지역 변수 예제
```kotlin
fun calculateAverage(numbers: List<Int>): Double {
    var sum = 0 // 지역 변수, 함수 내에서만 사용
    for (number in numbers) {
        sum += number
    }
    return sum.toDouble() / numbers.size
}
```

### 멤버 변수 최소화 예제
```kotlin
class SimpleCounter {
    private var count = 0 // 멤버 변수이지만 최소한으로 사용

    fun increment() {
        count++
    }

    fun getCurrentCount(): Int {
        return count
    }
}
```

## 핵심 요약
- 변수의 범위를 최소화하여 코드의 가독성과 유지보수성을 향상시킵니다.
- 필요한 범위 내에서만 변수를 사용하고, 지역 변수를 우선적으로 활용합니다.
- 클래스 멤버 변수는 꼭 필요한 경우에만 사용하여 클래스의 책임을 명확히 합니다.
