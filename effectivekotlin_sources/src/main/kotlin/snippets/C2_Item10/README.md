# Item 10: 단위 테스트 작성하기

단위 테스트는 코드의 특정 부분이 예상대로 작동하는지 검증하는 데 있어 핵심적인 역할을 합니다. 이를 통해 버그를 조기에 발견하고 코드의 신뢰성을 높일 수 있습니다. 다음은 Kotlin에서 효과적인 단위 테스트를 작성하기 위한 베스트 프랙티스와 함께 이해하기 쉬운 가이드입니다.

## 핵심 가이드라인 및 이유 (Best Practice)

### 1. **테스트 케이스 명확화**
   - **가이드라인**: 각 테스트 케이스는 하나의 기능만 검증해야 합니다. 이렇게 하면 문제 발생 시 원인을 더 쉽게 파악할 수 있습니다.
   - **이유**: 단일 책임 원칙을 따름으로써 코드 유지보수와 디버깅이 훨씬 용이해집니다.

### 2. **예상 값과 실제 값 비교**
   - **가이드라인**: 테스트 케이스에서 함수의 출력값을 명시된 예상 값과 비교해야 합니다.
   - **이유**: 이는 코드의 동작이 기대대로 이루어지고 있는지 객관적으로 확인할 수 있게 해줍니다.

### 3. **Mock 객체 사용**
   - **가이드라인**: 외부 의존성(예: 데이터베이스 접근, 네트워크 호출 등)을 가진 클래스의 경우, 실제 호출 대신 Mock 객체를 사용하여 테스트의 독립성을 유지하세요.
   - **이유**: 실제 환경에 의존하지 않아 테스트가 더 빠르고 일관성 있게 실행될 수 있습니다.

### 4. **테스트 범위 확장**
   - **가이드라인**: 기능 단위 테스트 외에도 경계 조건 및 예외 상황에 대한 테스트를 포함하세요.
   - **이유**: 다양한 입력에 대한 견고성을 검증하여 소프트웨어의 안정성을 향상시킵니다.

## 예제 코드

아래는 간단한 Kotlin 클래스와 그에 대한 단위 테스트 예제입니다.

### 클래스 예제
```kotlin
class Calculator {
    fun add(a: Int, b: Int): Int {
        return a + b
    }
}
```

### 단위 테스트 예제
```kotlin
import org.junit.jupyter.Assert.assertEquals
import org.junit.Test

class CalculatorTest {
    private val calculator = Calculator()

    @Test
    fun testAddPositiveNumbers() {
        // 예상 결과와 실제 결과 비교
        val result = calculator.add(2, 3)
        assertEquals(5, result, "두 양수를 더한 결과가 올바르지 않음")
    }

    @Test
    fun testAddWithNegativeNumbers() {
        // 경계 조건 테스트
        val result = calculator.add(-2, -3)
        assertEquals(-5, result, "음수를 더한 결과가 올바르지 않음")
    }
}
```

## 친근하고 전문적인 설명
이 가이드를 통해 단위 테스트의 중요성과 실제 작성 방법을 살펴보았습니다. 단위 테스트는 개발 과정에서 버그를 빠르게 찾아내고 코드의 신뢰성을 높이는 데 필수적입니다. 특히 Mock 객체를 활용하면 외부 의존성을 최소화하여 테스트의 효율성을 극대화할 수 있습니다. 다양한 시나리오를 테스트함으로써 소프트웨어의 견고성을 확보할 수 있으니, 이를 적극 활용해보세요!
