## 왜 함수 리터럴을 사용해야 할까요?

익명 함수는 특정 상황에서만 일시적으로 사용되는 짧은 함수를 정의할 때 유용합니다. 전통적인 방식으로는 함수를 정의하고 이름을 부여해야 하지만, 함수 리터럴을 사용하면 코드를 훨씬 간결하게 만들 수 있습니다. 특히 고차 함수(Higher-Order Function)와 함께 사용될 때 빛을 발하는데, 고차 함수는 다른 함수를 인자로 받거나 함수를 반환하는 함수를 말합니다.

함수 리터럴을 사용하는 주된 이유는 다음과 같습니다:

*   **가독성 향상:** 짧은 함수를 별도로 정의하지 않고 바로 사용하므로 코드의 흐름을 더 쉽게 파악할 수 있습니다.
*   **코드 간결성:** 함수 정의와 사용을 한 곳에서 처리하므로 코드의 양을 줄일 수 있습니다.
*   **유연성:** 고차 함수와 함께 사용할 때 특정 로직을 쉽게 인자로 전달할 수 있습니다.

## 함수 리터럴의 기본 문법

함수 리터럴은 다음과 같은 형태로 정의됩니다:

```kotlin
{ 파라미터 목록 -> 함수 본문 }
```

*   `{ }`: 중괄호로 함수 리터럴을 감쌉니다.
*   `파라미터 목록`: 함수의 파라미터를 정의합니다.  파라미터가 없다면 생략 가능합니다.
*   `->`: 파라미터 목록과 함수 본문을 구분합니다.
*   `함수 본문`: 함수가 수행할 작업을 정의합니다.

## 함수 리터럴 사용 예시

몇 가지 예시를 통해 함수 리터럴을 어떻게 사용하는지 살펴볼게요.

**1. `map` 함수와 함께 사용하기:**

`map` 함수는 컬렉션의 각 요소에 주어진 함수를 적용하여 새로운 컬렉션을 생성합니다.

```kotlin
val numbers = listOf(1, 2, 3, 4, 5)

// 각 숫자를 두 배로 만드는 함수 리터럴
val doubledNumbers = numbers.map { it * 2 }

println(doubledNumbers) // Output: [2, 4, 6, 8, 10]
```

위 예제에서 `{ it * 2 }`는 함수 리터럴입니다. `it`은 `map` 함수에서 전달되는 각 요소를 나타냅니다.

**2. `filter` 함수와 함께 사용하기:**

`filter` 함수는 주어진 조건을 만족하는 요소만 남겨 새로운 컬렉션을 생성합니다.

```kotlin
val numbers = listOf(1, 2, 3, 4, 5)

// 짝수만 필터링하는 함수 리터럴
val evenNumbers = numbers.filter { it % 2 == 0 }

println(evenNumbers) // Output: [2, 4]
```

여기서 `{ it % 2 == 0 }`는 짝수를 판별하는 함수 리터럴입니다.

**3. 인라인 함수 정의:**

함수 리터럴은 간단한 인라인 함수를 정의하는 데에도 사용할 수 있습니다.

```kotlin
fun processNumber(number: Int, operation: (Int) -> Int): Int {
  return operation(number)
}

val result = processNumber(5) { it * it } // 제곱하는 함수 리터럴
println(result) // Output: 25
```

`processNumber` 함수는 다른 함수를 인자로 받습니다. 이 때 함수 리터럴을 사용하여 간단하게 제곱 연산을 정의하고 인자로 전달할 수 있습니다.

## 타입 추론과 함수 리터럴

코틀린은 타입 추론 기능을 제공하므로, 함수 리터럴의 파라미터 타입이나 반환 타입을 명시적으로 지정하지 않아도 컴파일러가 자동으로 추론해줍니다.  위 예제에서 `(Int) -> Int` 처럼 타입을 명시하지 않아도 코틀린은 자동으로 타입을 추론합니다.

## 핵심 요약

*   함수 리터럴은 이름 없는 함수를 간결하게 정의하는 방법입니다.
*   `map`, `filter` 등의 고차 함수와 함께 사용하면 코드 가독성과 간결성을 높일 수 있습니다.
*   코틀린의 타입 추론 기능을 활용하여 함수 리터럴을 더욱 간결하게 사용할 수 있습니다.
*   함수 리터럴은 짧고 일시적인 함수 로직을 표현하는 데 효과적입니다.
