# 22. Testing

안녕하세요! 코틀린 프로그래밍을 처음 시작하시는 여러분께, 테스트의 중요성과 `Atomic Kotlin` 책에서 소개하는 간단한 테스트 프레임워크에 대해 쉽게 설명해드릴게요. 테스트는 프로그램 개발을 빠르고 안정적으로 진행하는 데 정말 필수적인 요소랍니다. 그럼 함께 살펴보도록 하죠!

## 테스트의 중요성

코드를 조금만 수정해도 다른 부분에 문제가 생기는 경우, 테스트 덕분에 문제를 바로 찾아낼 수 있어요. 만약 바로 알려주지 않으면 변경 사항들이 누적되어 문제 원인을 파악하는 데 훨씬 더 많은 시간이 걸릴 수 있답니다. 그래서 테스트는 프로그래밍 과정에서 꼭 필요한 부분이에요. 책 전체를 통해 테스트를 적극적으로 사용해보며, 테스트를 일상적인 프로그래밍 습관으로 만들어보세요.

### 간단한 테스트 방법

단순히 `println()`을 사용해 코드의 정확성을 확인하는 것은 한계가 있어요. 매번 출력 결과를 꼼꼼히 확인해야 하거든요. 그래서 이 책에서는 여러분이 좀 더 편하게 테스트를 경험할 수 있도록 간단한 테스트 시스템을 만들어봤어요. 이 시스템의 주요 목표는 다음과 같습니다:

1. **예상 결과 확인**: 표현식의 예상 결과를 보여줍니다.
2. **실행 확인**: 테스트가 성공하면 프로그램이 제대로 실행되고 있는지 확인할 수 있는 출력을 제공합니다.
3. **조기 교육**: 테스트를 일찍부터 자연스럽게 익히도록 합니다.

### 예제 코드 살펴보기

이제 실제 코드 예시를 통해 어떻게 사용하는지 보겠습니다.

#### 기본 테스트 예제

```kotlin
// Testing/TestingExample.kt
import atomictest.*

fun main() {
    val v1 = 11          // 정수 변수 v1
    val v2 = "Ontology"  // 문자열 변수 v2

    // `eq`은 "같음"을 의미합니다
    v1 eq 11           // 예상 결과: 11
    v2 eq "Ontology"   // 예상 결과: Ontology

    // `neq`은 "다름"을 의미합니다
    v2 neq "Epistimology" // 오류 예시: Epistimology와 비교
    // v2 eq "Epistimology" // 이 줄을 주석 해제하면 오류 메시지가 나옵니다

    /* 출력 예시:
    11
    Ontology
    Ontology
    */
}
```

#### `trace` 기능 사용 예제

```kotlin
// Testing/Trace1.kt
import atomictest.*

fun main() {
    trace("line 1")       // 출력: line 1
    trace(47)             // 출력: 47
    trace("line 2")       // 출력: line 2
    trace eq """
    line 1
    47
    line 2
    """ // 출력 확인
}
```

### 핵심 개념 요약

1. **`eq`과 `neq`**:
   - `eq`: 표현식의 결과가 예상 값과 같은지 확인합니다.
   - `neq`: 표현식의 결과가 예상 값과 다른지 확인합니다.
   - 다양한 타입의 값을 비교할 수 있으며, 문자열인 경우 문자열 비교를, 숫자인 경우 숫자 비교를 합니다.

2. **테스트 실패 시**:
   - 예상과 실제 값이 다르면 `[Error]` 메시지와 함께 오류 메시지가 출력됩니다.
   - 실제 값과 예상 값이 어떻게 다른지 명확히 보여줍니다.

3. **`trace`**:
   - 코드 실행 중에 출력을 추적하고 나중에 비교할 수 있게 해줍니다.

이 간단한 테스트 프레임워크를 통해 코드의 정확성을 쉽게 확인하고, 테스트의 중요성을 체감하실 수 있을 거예요. 책을 계속 읽으며 더 다양한 테스트 기법을 배워나가세요! 궁금한 점이 있으면 언제든지 물어봐주세요. 열심히 따라오시면 분명 큰 도움이 될 거예요!
