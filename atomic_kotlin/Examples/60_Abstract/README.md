# 60. Abstract

안녕하세요! 이번 챕터에서는 **추상 클래스**와 **추상 메서드**에 대해 배워볼게요. 처음 듣는 단어들이라 낯설 수도 있지만, 쉽게 이해할 수 있도록 차근차근 설명해드릴게요.

## 무엇이 추상 클래스인가요?

추상 클래스는 자식 클래스들이 반드시 구현해야 할 일부 메서드를 선언해두는 클래스예요. 쉽게 말해, **기본 틀**을 제공하면서 일부 부분은 자식 클래스들이 채워 넣어야 하는 공간이 있는 클래스라고 생각하면 돼요.

### 핵심 개념 요약
- **추상 클래스**는 직접 인스턴스화할 수 없어요. 주로 하위 클래스를 정의하는 데 사용됩니다.
- 추상 클래스에는 **추상 메서드**가 포함될 수 있어요. 추상 메서드는 `abstract` 키워드로 선언되며, 본문이 없이 선언만 하는 메서드예요.

## 추상 메서드 예시

아래 예제를 통해 추상 메서드가 어떻게 작동하는지 확인해볼게요.

```kotlin
// 추상 클래스 예시
abstract class Animal {
    // 추상 메서드 선언
    abstract fun makeSound(): String  // 본문이 없는 메서드

    // 일반 메서드 예시
    fun sleep() {
        println("Sleep is good...")
    }
}

// 추상 클래스를 상속받는 자식 클래스 예시
class Dog : Animal() {
    // 추상 메서드 구현
    override fun makeSound(): String {
        return "Woof woof!"
    }
}

// 또 다른 자식 클래스 예시
class Cat : Animal() {
    // 추상 메서드 구현
    override fun makeSound(): String {
        return "Meow meow!"
    }
}

// 사용 예시
fun main() {
    Dog().sleep()       // 출력: Sleep is good...
    println(Dog().makeSound())  // 출력: Woof woof!

    Cat().sleep()       // 출력: Sleep is good...
    println(Cat().makeSound())  // 출력: Meow meow!
}
```

### 설명
1. **Animal 클래스**는 추상 클래스로, `makeSound()`라는 추상 메서드를 정의하고 있어요. 이 메서드는 자식 클래스에서 반드시 구현해야 해요.
2. **Dog 클래스**와 **Cat 클래스**는 `Animal` 클래스를 상속받아서 `makeSound()` 메서드를 각각의 특징에 맞게 구현하고 있어요.
3. **main 함수**에서는 각 클래스의 메서드를 호출하여 결과를 출력해요. `sleep()` 메서드는 모든 `Animal` 클래스의 하위 클래스에서 공통적으로 사용할 수 있는 일반 메서드예요.

## 왜 추상 클래스가 필요한가요?

- **공통 인터페이스 제공**: 여러 하위 클래스가 공유할 공통 인터페이스를 정의할 수 있어요.
- **타입 안전성**: 추상 클래스를 상속받는 객체들끼리 일관성 있게 동작하도록 보장해요.
- **코드 재사용성**: 공통적인 동작을 정의해두고, 자식 클래스에서 필요한 부분만 다르게 구현할 수 있어요.

이렇게 추상 클래스와 추상 메서드를 활용하면 코드의 구조를 더 명확하게 만들고, 유연성과 확장성을 높일 수 있어요. 궁금한 점이 있으면 언제든지 물어보세요!
