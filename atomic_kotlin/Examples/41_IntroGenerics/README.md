# 41. 제네릭(Generics) 소개하기

안녕하세요! 코틀린 프로그래밍을 처음 배우는 분들을 위해 제네릭(Generics)에 대해 친절하게 설명해드릴게요. 제네릭은 코드를 더 유연하고 재사용 가능하게 만드는 강력한 기능이에요. 지금부터 쉽게 이해할 수 있도록 하나씩 살펴보겠습니다.

## 기본 개념

### 무엇이 제네릭인가요?
제네릭은 특정 타입을 명시하지 않고도 코드를 작성할 수 있게 해주는 코틀린의 특징입니다. 쉽게 말해, 함수나 클래스가 여러 타입을 처리할 수 있도록 설계된 거죠. 이렇게 하면 코드를 한 번 작성해도 다양한 데이터 타입에 적용할 수 있어요.

### 왜 제네릭을 사용할까요?
- **유연성**: 다양한 타입을 처리할 수 있어요.
- **재사용성**: 한 번 작성한 코드를 여러 타입에 적용할 수 있어요.
- **안전성**: 컴파일 타임에 타입 오류를 잡아낼 수 있어요.

## 기본 예제

### 제네릭 클래스 예시
아래는 간단한 제네릭 클래스 예시예요. `Box` 클래스는 어떤 타입의 데이터든 저장할 수 있어요.

```kotlin
class Box<T> {
    private var content: T? = null

    fun set(content: T) {
        this.content = content
    }

    fun get(): T? {
        return content
    }
}

// 사용 예시
fun main() {
    val intBox = Box<Int>()  // Int 타입의 Box 생성
    intBox.set(42)          // 정수 42를 저장
    println(intBox.get())   // 출력: 42

    val stringBox = Box<String>()  // String 타입의 Box 생성
    stringBox.set("Hello")        // 문자열 "Hello"를 저장
    println(stringBox.get())      // 출력: Hello
}
```

### 설명
- `Box<T>`: 여기서 `T`는 제네릭 타입 매개변수를 나타내요. `T` 대신 `Int`, `String`, `Double` 등 특정 타입을 직접 지정할 수 있어요.
- `content: T?`: `content`는 `T` 타입의 값을 저장할 수 있어요. 널(null)을 허용하기 위해 `?`를 사용해요.
- `set(content: T)`와 `get()`: 함수는 저장하고 가져오는 기능을 수행해요. 타입이 자동으로 결정되어요.

## 주요 핵심 개념 요약
1. **타입 매개변수**: 제네릭에서 `T`와 같은 타입 매개변수를 사용해 특정 타입을 명시하지 않고도 코드를 작성할 수 있어요.
2. **타입 안전성**: 컴파일 타임에 타입 오류를 잡아낼 수 있어요. 런타임 오류를 미리 방지할 수 있죠.
3. **다양한 타입 적용**: 한 개의 클래스로 여러 타입의 데이터를 처리할 수 있어요. 예를 들어, `Box` 클래스는 정수든 문자열이든 어떤 타입이든 저장할 수 있어요.

이렇게 제네릭을 사용하면 코드의 유연성과 재사용성이 크게 향상되어요. 이제 여러분도 다양한 타입의 데이터를 처리하는 코틀린 코드를 쉽게 작성할 수 있을 거예요! 궁금한 점이 있으면 언제든지 물어보세요. 함께 배우는 재미로 코딩해봅시다!
