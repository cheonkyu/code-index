# 77. Logging

안녕하세요! 오늘은 코틀린 프로그래밍에서 매우 중요한 주제인 로깅(Logging)에 대해 배워볼게요. 로깅은 프로그램이 실행되는 동안 중요한 정보를 기록하는 기능을 말해요. 쉽게 말해, 프로그램이 어떤 작업을 수행하고 있는지, 어떤 문제가 발생했는지 등을 기록하는 거죠. 이걸 통해 개발자들은 문제 해결이나 프로그램 동작 파악에 큰 도움을 받을 수 있어요.

## 로깅의 실제 사례

- **설치 프로그램**: 설치 과정 중에 어떤 단계를 거쳤는지, 파일 저장을 위한 디렉토리는 어디인지, 프로그램 시작 시의 값 등을 기록해요.
- **웹 서버**: 각 요청의 출처 주소와 상태를 기록해요.

### 로깅의 장점

1. **디버깅 도움**: 프로그램이 잘 작동할 때 `println()`을 사용해 임시로 상태를 확인할 수 있어요. 하지만 실제로 프로덕션 환경에 적용할 때는 이런 임시 코드를 제거하는 것이 보통이죠.
2. **동적 제어**: 로깅은 필요할 때 켜고 필요 없을 때는 끄는 것이 가능해요. 이는 프로그램 성능에 영향을 덜 미치면서 유연하게 사용할 수 있게 해줍니다.
3. **오류 추적**: 프로그램이 오류를 겪고도 계속 실행될 수 있도록 돕습니다. 오류 발생 시 자세한 정보를 기록해 나중에 분석할 수 있어요.

## 코틀린에서 로깅하기: Kotlin Logging 사용하기

코틀린에서는 `Kotlinlogging` 라이브러리를 주로 사용해요. 이 라이브러리는 코틀린의 감각과 간결함을 반영하고 있어요. 다른 로깅 패키지들도 많이 있지만, `Kotlinlogging`은 사용하기 편하답니다.

### 기본적인 로깅 예제

아래는 `Kotlinlogging`을 사용한 간단한 예제 코드예요. 이 코드는 파일 스코프에서 로거를 생성하고 다양한 로깅 레벨을 보여줍니다.

```kotlin
// Logging/BasicLogging.kt
package logging

import mu.KLogging

// 로거 객체 생성
private val log = KLogging().logger

fun main() {
    val msg = "안녕하세요, 코틀린 로깅!"
    
    // 다양한 로깅 레벨 사용 예시
    log.trace(msg)      // 상세한 추적 정보 (디버그용)
    log.debug(msg)      // 디버깅용 정보
    log.info(msg)       // 일반적인 정보 기록
    log.warn(msg)       // 경고 메시지
    log.error(msg)      // 오류 메시지
    
    // 주의: trace와 debug 레벨은 기본 설정에서 출력되지 않을 수 있어요.
    // 설정을 변경해야 출력됩니다.
}
```

### 주요 로깅 레벨 요약

- **trace**: 매우 세밀한 추적 정보 (주로 디버깅 용도)
- **debug**: 디버깅 정보 (개발 중에 자주 사용)
- **info**: 일반적인 프로그램 동작 정보 (정상적인 작동 상태 확인)
- **warn**: 경고 메시지 (문제가 발생할 수 있음을 알림)
- **error**: 오류 메시지 (심각한 문제 발생)

### 로깅 설정 변경하기

기본 설정에서는 `trace`와 `debug` 레벨의 메시지가 출력되지 않을 수 있어요. 다른 로깅 레벨을 사용하려면 로깅 설정을 조정해야 합니다. 이 설정은 사용하는 로깅 패키지에 따라 다르므로, 자세한 내용은 해당 패키지의 문서를 참고하세요.

### 로깅 패키지의 역사와 구조

로깅 라이브러리는 오랜 시간 동안 발전해왔어요:
- 초기 Java 로깅 라이브러리의 한계로 새로운 라이브러리들이 생겨났어요.
- 이를 통합하기 위해 공통 인터페이스들이 만들어졌고, 이를 `facade`라고 부르는데, 여러 로깅 프레임워크를 통합하는 역할을 해요.
- `Kotlinlogging`은 `SLF4J`라는 Java 기반 로깅 프레임워크의 `facade` 위에 세워져 있어요. `SLF4J`는 여러 로깅 프레임워크를 감싸서 사용하기 쉽게 만들어줘요.

### 간단 요약

- **로깅은 프로그램의 동작을 추적하고 문제를 해결하는 데 필수적입니다.**
- **`Kotlinlogging`을 사용하면 코틀린 코드에 자연스럽게 통합할 수 있어요.**
- **로깅 레벨을 조절하여 필요한 정보만 기록할 수 있어요.**
