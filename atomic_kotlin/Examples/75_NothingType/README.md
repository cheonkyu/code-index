# 75. NothingType

안녕하세요! 이번 챕터에서는 Kotlin에서 조금 특별한 타입인 `Nothing`에 대해 배워볼게요. `Nothing` 타입은 초보자분들에게는 처음 접할 때 조금 당황스러울 수 있지만, 실제로 코딩하면서 이해하면 매우 유용한 개념이에요. 그럼 함께 알아보도록 합시다!

## 기본 개념 이해하기

`Nothing` 타입은 사실상 Kotlin에서 존재하는 가장 기본적이면서도 추상적인 타입 중 하나예요. 이름 그대로 '아무것도 아닌 타입'이라고 생각하면 됩니다. 주요 특징은 다음과 같아요:

- **반환 타입**: 함수가 어떤 예외를 던져도 안전하게 실행되지 않을 때 반환 타입으로 사용되요. 예를 들어, 프로그램이 중지될 가능성이 있는 상황에서는 `Nothing`을 반환 타입으로 지정할 수 있어요.
  
- **비교 불가능**: `Nothing`은 다른 어떤 타입과도 비교할 수 없어요. 즉, `is` 연산자나 `==` 연산자로 비교할 수 없답니다.

### 예제 코드 살펴보기

아래 예제 코드를 통해 `Nothing` 타입이 어떻게 사용되는지 살펴보도록 해요.

```kotlin
// 함수가 실행 불가능한 상황을 표현할 때 Nothing 반환
fun riskyOperation(): Nothing = throw IllegalArgumentException("위험한 작업으로 인해 프로그램이 중단될 수 있습니다.")

// Nothing 타입을 사용한 함수 호출 예시
try {
    riskyOperation()
} catch (e: Nothing) {
    println("프로그램이 안전하게 중지되었습니다: ${e}")
}

// Nothing을 직접 사용해보는 예제
val value: Nothing = null // Nothing 타입 변수는 null과 함께 사용될 수 있어요.
```

#### 설명

1. **riskyOperation 함수**:
   - 이 함수는 `throw IllegalArgumentException`을 사용해 예외를 던집니다. 이 경우 함수는 `Nothing` 타입을 반환하게 되어 있어요. 즉, 이 함수가 실행되면 프로그램이 중단될 수 있다는 경고 메시지를 보여줍니다.

2. **try-catch 블록**:
   - `catch` 블록에서 `Nothing` 타입을 잡아서 안전하게 프로그램이 중지되도록 처리하고 있어요. 이렇게 하면 예외 처리가 더 명확해집니다.

3. **Nothing 타입 변수**:
   - `Nothing` 타입 변수는 사실상 `null`처럼 사용될 수 있어요. 하지만 `Nothing` 타입 변수는 특별한 의미를 지니기 때문에 주의해서 사용해야 합니다.

## 핵심 요약

- **Nothing 타입**은 프로그램이 안전하게 중단되거나 실행 불가능한 상황을 표현할 때 사용됩니다.
- `Nothing`은 다른 타입과 비교할 수 없으며, 주로 예외 처리나 함수의 반환 타입으로 활용됩니다.
- 실제 코딩에서는 예외 처리를 명확히 하거나 프로그램의 안전성을 강조할 때 유용하게 사용됩니다.
