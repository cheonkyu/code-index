# 80. Scope Functions

안녕하세요, 코틀린 초보자 여러분! 이번 챕터에서는 **Scope Functions**에 대해 배워볼 거예요. Scope Functions는 코드를 더 간결하고 읽기 쉽게 만드는 데 도움을 주는 특별한 함수들입니다. 이름 그대로 특정 범위 내에서만 사용되며, 추가적인 기능을 제공하는 것이 아니라 코드의 가독성을 높이는 역할을 해요. 주요 Scope Functions는 `let()`, `run()`, `with()`, `apply()`, 그리고 `also()`가 있어요. 각각의 함수는 사용 방식과 맥락에 따라 조금씩 다르게 작동하니, 하나씩 살펴볼게요!

## 주요 Scope Functions 개요

### 1. **let()**
- **목적**: 임시 변수를 사용하지 않고 객체에 접근할 수 있게 해줍니다.
- **접근 방법**: `it`이라는 이름으로 객체를 참조합니다.
- **특징**: 람다 표현식 내의 마지막 표현식을 반환합니다.
- **예제 코드**:
  ```kotlin
  data class Tag(var n: Int = 0, var s: String = "")

  fun main() {
      // 기본 사용법
      Tag(1).let {
          it.s = "let: ${it.n}"  // 객체에 접근 및 수정
          it.n++          // 마지막 표현식 실행
      } eq 2  // 예상 결과: Tag(n=2)

      // 명명된 람다 인자 사용
      Tag(2).let { tag ->
          tag.s = "let: ${tag.n}"
          tag.n++
      } eq 3  // 예상 결과: Tag(n=3)
  }
  ```

### 2. **run()**
- **목적**: 객체를 임시로 사용할 때 명확한 컨텍스트를 제공합니다.
- **접근 방법**: `this`를 사용하여 객체에 접근합니다.
- **특징**: 람다 표현식 내의 마지막 표현식을 반환합니다.
- **예제 코드**:
  ```kotlin
  fun main() {
      Tag(3).run {
          s = "run: $n"  // this를 통해 객체 접근
          n++
      } eq 4  // 예상 결과: Tag(n=4)
      // 이 경우 내부 `n`이 변경됩니다.
  }
  ```

### 3. **with()**
- **목적**: 객체의 컨텍스트 내에서 작업을 수행할 때 사용합니다.
- **접근 방법**: `this`를 통해 객체를 참조합니다.
- **특징**: 람다 표현식 내의 마지막 표현식을 반환합니다.
- **예제 코드**:
  ```kotlin
  fun main() {
      with(Tag(4)) {
          s = "with: $n"  // this를 통해 객체 접근
          n++
      } eq 5  // 예상 결과: Tag(n=5)
  }
  ```

### 4. **apply()**
- **목적**: 객체 자체를 수정하고 결과로 반환합니다.
- **접근 방법**: `this`를 통해 객체에 접근합니다.
- **특징**: 수정된 객체를 반환합니다.
- **예제 코드**:
  ```kotlin
  fun main() {
      val tag = Tag(5).apply {
          s = "apply: $n"  // 객체 수정
          n++
      }
      eq("Tag(n=6)", tag)  // 예상 결과: Tag(n=6) 객체 반환
  }
  ```

### 5. **also()**
- **목적**: 객체를 임시로 사용하면서 수정할 때 유용합니다.
- **접근 방법**: `it`을 통해 객체를 참조하고, 명명된 람다 인자를 사용할 수 있습니다.
- **특징**: 수정된 객체를 반환합니다.
- **예제 코드**:
  ```kotlin
  fun main() {
      val tag = Tag(6).also {
          it.s = "also: ${it.n}"  // 객체 수정
          it.n++
      }
      eq("Tag(n=7)", tag)  // 예상 결과: Tag(n=7) 객체 반환
  }

  // 명명된 람다 인자 사용 예제
  fun main() {
      val tag = Tag(7).also { tag ->
          tag.s = "also: ${tag.n}"
          tag.n++
      }
      eq("Tag(n=8)", tag)  // 예상 결과: Tag(n=8) 객체 반환
  }
  ```

## 특징 요약
- **`this` 사용**: `run()`, `with()`, `apply()`에서 주로 사용됩니다. 코드 블록 내에서 객체를 명확하게 참조합니다.
- **`it` 사용**: `let()`, `also()`에서 주로 사용됩니다. 명명된 람다 인자를 제공할 수 있어요.
- **반환 값**:
  - 마지막 람다 표현식을 반환하는 함수: `let()`, `run()`, `with()`
  - 수정된 객체를 반환하는 함수: `apply()`, `also()`

## null 안전성
Scope Functions를 사용할 때 null 안전성도 중요해요! `?.` 안전 액세스 연산자를 사용하면 null 체크를 한 번에 처리할 수 있어요. 예를 들어:

```kotlin
fun main() {
    val str?:= gets()  // gets()가 null이면 안전하게 처리
    str?.let {  // null 체크 후 let() 사용
        println(it.removeSuffix("!") + it.length) eq "str4"
    }
}
```

이렇게 하면 `gets()`가 null일 경우 전체 Scope Function 범위가 실행되지 않아 안전성을 확보할 수 있어요.
