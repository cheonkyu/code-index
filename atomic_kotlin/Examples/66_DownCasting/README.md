# 66. DownCasting (하강 캐스팅)

안녕하세요! 코틀린 프로그래밍을 처음 접하는 분들을 위해 오늘은 `Downcasting` 또는 하강 캐스팅에 대해 쉽게 설명해 드리겠습니다. 이 개념은 객체 지향 프로그래밍에서 자주 사용되며, 특히 객체의 타입을 유연하게 다루는 데 도움이 됩니다. 지금부터 차근차근 이해해 보도록 합시다!

## 핵심 개념 요약

- **하강 캐스팅(Downcasting)**: 부모 클래스의 참조 변수를 자식 클래스의 객체로 변환하는 과정입니다. 코틀린에서는 `is` 키워드와 `as` 키워드를 사용해 이를 수행합니다.
- **목적**: 특정 객체가 특정 하위 타입인지 확인하고, 그에 따라 타입을 변환하여 더 구체적인 메소드나 속성에 접근할 수 있게 합니다.
- **안전성**: `as` 키워드를 사용할 때는 반드시 타입 체크를 통해 안전하게 변환해야 합니다. 잘못된 캐스팅은 컴파일 오류를 일으킬 수 있습니다.

## 기본 예시

### 예제 코드

아래 코드는 `Animal` 클래스의 하위 클래스인 `Dog` 객체를 다루는 예시입니다.

```kotlin
// 상위 클래스 정의
open class Animal {
    open fun speak() {
        println("동물 소리")
    }
}

// 하위 클래스 정의
class Dog : Animal() {
    override fun speak() {
        println("멍멍!")
    }
}

// 하위 클래스 객체 생성
val animal: Animal = Dog()

// Downcasting 예시
if (animal is Dog) {  // 타입 체크
    val speakingDog = animal as Dog  // 안전한 캐스팅
    speakingDog.speak()  // "멍멍!" 출력
} else {
    animal.speak()  // 상위 클래스 메소드 호출, "동물 소리" 출력
}
```

### 설명

1. **상위 클래스와 하위 클래스 정의**:
   - `Animal`은 상위 클래스로 기본적인 `speak` 메소드를 제공합니다.
   - `Dog`는 `Animal`의 하위 클래스로 `speak` 메소드를 오버라이드하여 고유한 소리를 출력합니다.

2. **하강 캐스팅 과정**:
   - `animal` 변수는 `Animal` 타입으로 선언되었지만, 실제로는 `Dog` 객체입니다.
   - `if (animal is Dog)`로 타입 체크를 수행하여 `animal`이 `Dog`인지 확인합니다.
   - 타입 체크가 성공하면 `animal as Dog`로 안전하게 캐스팅하여 `Dog` 타입의 메소드를 호출할 수 있습니다.

3. **안전성 주의사항**:
   - 타입 체크 없이 `as`를 사용하면 컴파일 오류가 발생할 수 있으므로 항상 타입 검증을 해야 합니다.

## 간단 요약

하강 캐스팅은 부모 클래스 객체를 자식 클래스 객체로 변환하는 방법입니다. 이를 통해 더 구체적인 타입의 기능을 사용할 수 있지만, 반드시 타입 체크를 통해 안전성을 확보해야 합니다. 이제 이 개념을 연습하면서 코틀린 프로그래밍의 유연성을 체험해 보세요!
